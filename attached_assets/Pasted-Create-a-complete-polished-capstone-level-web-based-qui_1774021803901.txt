Create a complete, polished, capstone-level web-based quiz game using only HTML, CSS, and JavaScript. Do not use frameworks, backend, APIs, or database. Everything must run fully in the browser only.

PROJECT GOAL:
Build a highly polished, colorful, interactive, game-like quiz application inspired by the energy and excitement of party quiz games like Kahoot, but with an original design. The final result should feel like a real mini web game suitable for a capstone project presentation.

GENERAL RULES:
- Use only plain HTML, CSS, and JavaScript
- No frameworks
- No libraries
- No backend
- No database
- No build tools
- No installation required
- Must run directly in browser
- Code must be beginner-friendly, readable, and editable in Replit or VS Code

CORE GAME FLOW:
The app must include the following screens and states:
1. Start Screen
2. Instructions / How to Play section
3. Quiz Screen
4. Answer Feedback state
5. Final Results Screen
6. High Score display
7. Restart / Play Again flow

START SCREEN REQUIREMENTS:
- Show a large game title
- Show a short subtitle or tagline
- Show a short “How to Play” instruction box
- Show current saved high score
- Show a Start Quiz button
- Add a visually exciting hero-style layout
- Add decorative animated background shapes or floating elements for game feel

QUIZ SCREEN REQUIREMENTS:
- Show one question at a time
- Each question must have exactly 4 multiple-choice answers
- Questions should appear in a centered card or panel
- Show question number clearly (example: Question 3 of 10)
- Show current score while playing
- Show a visible countdown timer
- Show a progress bar for quiz completion
- Show category or difficulty label if possible
- Keep the layout responsive and clean

QUESTION DATA:
- Use at least 10 questions
- Store them in a JavaScript array of objects
- Each object should include:
  - question text
  - 4 answer choices
  - correct answer
  - optional category
  - optional difficulty
- Use a fun and presentation-friendly theme such as mixed trivia, pop culture, technology, gaming, anime, or student-friendly general knowledge

ANSWER INTERACTION:
- When a user clicks an answer:
  - disable all answer buttons immediately
  - highlight the selected answer
  - if correct, show green
  - if wrong, show red
  - also reveal the correct answer in green
- Add visual feedback such as:
  - glow effect
  - slight shake effect for wrong answers
  - bounce or pulse effect for correct answers
- Show a short feedback message such as:
  - Correct!
  - Nice try!
  - Time’s up!
- After answering, show a Next Question button
- Prevent multiple answer submissions

TIMER FEATURE:
- Add a countdown timer per question (for example 15 seconds)
- Display timer clearly and prominently
- Add visual urgency when time is low, such as:
  - timer color change
  - pulse animation
  - warning effect
- If timer reaches zero:
  - mark question as unanswered
  - disable all answers
  - reveal the correct answer
  - show “Time’s Up!”
  - enable Next Question button
- Reset timer for every new question

SCORING SYSTEM:
- Give 1 point for every correct answer
- Do not add points for wrong or unanswered questions
- Show live score while playing
- On final screen, show:
  - total score
  - total correct answers
  - total wrong answers
  - total unanswered questions
  - percentage score
- Show a performance remark depending on score, such as:
  - Excellent!
  - Great Job!
  - Good Effort!
  - Try Again!

HIGH SCORE FEATURE:
- Save high score using localStorage
- Display high score on the start screen
- Display high score on the result screen
- Update high score automatically if player beats previous best

RESULTS SCREEN:
- Show final score prominently
- Show percentage result
- Show performance message
- Show summary statistics:
  - correct
  - wrong
  - unanswered
- Show high score
- Add buttons for:
  - Play Again
  - Back to Start
- Add a review summary section if possible

OPTIONAL BUT HIGHLY RECOMMENDED CAPSTONE FEATURES:
Add as many of these as possible:
1. Category label per question
2. Difficulty label
3. Randomized question order each playthrough
4. Randomized answer button order
5. Sound effect placeholders in code comments
6. Simple entrance/exit animations between questions
7. Loading animation before quiz starts
8. Pause button
9. Mute/unmute UI button placeholder
10. Theme toggle (light/dark or neon mode)
11. Review Answers screen after finishing
12. Show which questions were answered correctly or incorrectly
13. Confetti or celebration animation when score is high
14. Motivational feedback when score is low
15. Keyboard accessibility for answer selection
16. Mobile-first responsive adjustments
17. Replayability by shuffling questions

REVIEW ANSWERS FEATURE:
After the final score screen, include a section or separate screen where the player can review:
- each question
- the selected answer
- the correct answer
- whether it was correct, wrong, or unanswered
